# SQLD 시험범위 정리

## 윈도우 함수

- `ROWS` : 차이점
- `RANGE` : 같은 값
- `RANK` : 중복 건너뛰기
- `DEXSE RANK` : 중복을 건너뛰지 않음
- `PARTITION BY`
- `ORDER BY`

## 계층형 질의

- `PRIOR 자식 데이터 = 부모 데이터` : `부모데이터 = PRIOR 자식데이터`와 동일한 의미
- 부모에서 자식으로 가는 경우 순방향

## 절차형 PL/SQL

- `EXCEPTION` : 생략 가능
- `PROCEDURE` 
- `TRIGGER` : COMMIT 롤백이 안 된다
- `USER DEFINED FUNCTION` : 값이 반드시 나온다
